# Analysis-and-Design
CS 319 Portfolio Reflection – Analysis and Design

What was the problem you were solving in the projects for this course?

The challenge I was working on was how to develop a good user experience for Kiva borrowers and lenders within the constraints of diverse technology. For borrowers, I built an interface for an embedded device with a flip-phone form factor that would allow users to access loan information, request loans, and track repayments. I researched how a contemporary cloud-based system could visualize data on loans, borrowers, and repayment statistics to help lenders make smarter lending decisions. The problem was designing solutions that addressed the users’ needs and the technical constraints.

How did you approach the problem?

I took the challenge and worked through user stories and what would be the most crucial information each user would require. I analyzed Kiva’s existing data, borrower and lender requirements, and prioritized the data that would help users meet their goals. Next I generated wireframes and design recommendations focused on simplicity, usability and accessibility. It is crucial to understand how information is arranged and displayed since the user needs to be able to discover and understand the material he/she needs quickly without being overwhelmed.

How did you overcome any roadblocks you encountered while going through the activities or project?

The first problem was designing for a flip-phone embedded device with a small screen and restricted navigation possibilities. I had to choose the most important facts and cut out the extraneous. Another problem was combining the needs of users with Kiva’s objective and ethical guidelines. To address these problems I focused on user needs, simplified navigation, and designed the interface to display information in a straightforward and courteous way. By creating various design concepts and polishing the wireframes, I was able to build a solution that was both functional and user-friendly.

How has your work on this project expanded your approach to designing software and developing programs?

This project showed me that great software design means understanding the user first, and the technology second. I learned how to collect requirements, examine user workflows and determine what information consumers need the most. It also taught me how software may be adapted from older systems to newer platforms while still accommodating users who may have restricted access to modern technologies. Today, I am more focused on building solutions that are accessible, intuitive and solving genuine problems for users.

How has your work on this project evolved the way you create maintainable, readable, and adaptable solutions?

The need for preparation and documentation before development was highlighted by the project. That helped me think through how users would interact with the system and how future enhancements could be made by creating wireframes and design proposals. I discovered that good design, where user needs, workflows, and data priorities are effectively organized, results in manageable and adaptive software. Design with usability and scalability in mind and it will be easier to create in future, and easier to adjust as time goes on and requirements change .
